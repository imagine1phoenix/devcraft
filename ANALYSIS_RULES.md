# ANALYSIS_RULES.md — Rules-Based Signal Generation

> This document defines the deterministic scoring algorithm used to generate Buy/Hold/Sell signals. The LLM does NOT decide signals — this algorithm does. The LLM only explains the result in natural language.

---

## 1. Scoring Philosophy

Signals are generated by a **weighted multi-factor scoring model** that evaluates a stock across four dimensions:

1. **Valuation** (25% weight) — Is the stock priced fairly?
2. **Growth** (25% weight) — Is the company growing?
3. **Financial Health** (25% weight) — Is the balance sheet strong?
4. **Technical Momentum** (25% weight) — What does price action suggest?

Each dimension produces a sub-score from **0 to 100**. The final score is a weighted average.

---

## 2. Signal Thresholds

| Score Range | Signal | Label | Color |
|---|---|---|---|
| 80–100 | **Strong Buy** | Strongly Bullish | `#16a34a` (dark green) |
| 65–79 | **Buy** | Moderately Bullish | `#22c55e` (green) |
| 45–64 | **Hold** | Neutral | `#eab308` (amber) |
| 25–44 | **Sell** | Moderately Bearish | `#ef4444` (red) |
| 0–24 | **Strong Sell** | Strongly Bearish | `#dc2626` (dark red) |

**Confidence Range Calculation:**
- If 3 or 4 dimensions agree on direction: confidence = 70–85%
- If 2 dimensions agree: confidence = 50–65%
- If dimensions are split evenly: confidence = 30–45%
- Always display as a range, never a single number

## 3. Valuation Score (25%)

### Metrics Used

| Metric | Source Field | How to Score |
|---|---|---|
| **P/E Ratio** | `PERatio` | Compare to sector median. Below = bullish, Above = bearish |
| **Forward P/E** | `ForwardPE` | Compare to trailing P/E. Lower = bullish (earnings growing) |
| **PEG Ratio** | `PEGRatio` | < 1.0 = Very Bullish, 1.0–1.5 = Fair, > 2.0 = Overvalued |
| **P/S Ratio** | `PriceToSalesRatioTTM` | Compare to sector median |
| **P/B Ratio** | `PriceToBookRatio` | < 1.0 = Possible value, > 3.0 = Premium valuation |
| **EV/EBITDA** | `EVToEBITDA` | < 10 = Attractive, 10–15 = Fair, > 20 = Expensive |

### Scoring Logic

```typescript
function scoreValuation(data: FundamentalData, sectorMedian: SectorData): number {
  let score = 50; // Start neutral

  // P/E vs sector
  if (data.peRatio && sectorMedian.peRatio) {
    const peRatio = data.peRatio / sectorMedian.peRatio;
    if (peRatio < 0.7) score += 15;       // Significantly undervalued
    else if (peRatio < 0.9) score += 8;   // Moderately undervalued
    else if (peRatio > 1.5) score -= 15;  // Significantly overvalued
    else if (peRatio > 1.2) score -= 8;   // Moderately overvalued
  }

  // PEG Ratio
  if (data.pegRatio) {
    if (data.pegRatio < 0.5) score += 15;
    else if (data.pegRatio < 1.0) score += 10;
    else if (data.pegRatio < 1.5) score += 3;
    else if (data.pegRatio > 2.5) score -= 12;
    else if (data.pegRatio > 2.0) score -= 6;
  }

  // EV/EBITDA
  if (data.evToEbitda) {
    if (data.evToEbitda < 8) score += 10;
    else if (data.evToEbitda < 12) score += 5;
    else if (data.evToEbitda > 25) score -= 10;
    else if (data.evToEbitda > 18) score -= 5;
  }

  return Math.max(0, Math.min(100, score)); // Clamp 0–100
}
```

### Missing Data Handling
- If a metric is missing (null/undefined), skip that metric
- If ALL valuation metrics are missing, return `null` (not 50) — signal "Insufficient Data"
- Weight is redistributed to available dimensions

## 4. Growth Score (25%)

### Metrics Used

| Metric | Calculation | How to Score |
|---|---|---|
| **Revenue Growth (YoY)** | `(currentRevenue - priorRevenue) / priorRevenue` | >20% = Strong, 5–20% = Moderate, <0% = Declining |
| **EPS Growth (YoY)** | `(currentEPS - priorEPS) / abs(priorEPS)` | >25% = Strong, 5–25% = Moderate, <0% = Declining |
| **Revenue Growth Trend** | Slope of last 4 quarters | Accelerating = Bullish, Decelerating = Bearish |
| **Earnings Surprise** | Actual EPS vs Consensus | Beat = Bullish, Miss = Bearish |
| **Gross Margin Trend** | last 4 quarters | Expanding = Bullish, Contracting = Bearish |

### Scoring Logic

```typescript
function scoreGrowth(data: FinancialStatements): number {
  let score = 50;

  // Revenue Growth YoY
  const revenueGrowth = calculateYoYRevGrowth(data);
  if (revenueGrowth !== null) {
    if (revenueGrowth > 0.30) score += 20;       // 30%+ growth
    else if (revenueGrowth > 0.15) score += 12;   // 15–30% growth
    else if (revenueGrowth > 0.05) score += 5;    // 5–15% growth
    else if (revenueGrowth > 0) score += 0;        // 0–5% (stagnant)
    else if (revenueGrowth > -0.10) score -= 10;   // Slight decline
    else score -= 20;                               // Significant decline (>10%)
  }

  // EPS Growth YoY
  const epsGrowth = calculateYoYEPSGrowth(data);
  if (epsGrowth !== null) {
    if (epsGrowth > 0.25) score += 15;
    else if (epsGrowth > 0.10) score += 8;
    else if (epsGrowth > 0) score += 3;
    else if (epsGrowth > -0.15) score -= 8;
    else score -= 15;
  }

  // Revenue Acceleration
  const isAccelerating = checkRevenueAcceleration(data);
  if (isAccelerating === true) score += 10;
  else if (isAccelerating === false) score -= 5;

  return Math.max(0, Math.min(100, score));
}
```

## 5. Financial Health Score (25%)

### Metrics Used

| Metric | Source | How to Score |
|---|---|---|
| **Debt-to-Equity** | `balance_sheet` | < 0.5 = Strong, 0.5–1.5 = Moderate, > 2.0 = Risky |
| **Current Ratio** | `balance_sheet` | > 2.0 = Strong, 1.0–2.0 = Adequate, < 1.0 = Risky |
| **Quick Ratio** | `balance_sheet` | > 1.5 = Strong, 0.7–1.5 = Adequate, < 0.7 = Risky |
| **Free Cash Flow** | `cash_flow` | Positive = Good, Negative = Concerning |
| **FCF Margin** | FCF / Revenue | > 15% = Excellent, 5–15% = Good, < 0% = Bad |
| **Operating Margin** | `income_statement` | Compare to sector, higher = better |
| **Net Margin** | `income_statement` | Positive and stable = Bullish |
| **Return on Equity (ROE)** | `overview` | > 20% = Excellent, 10–20% = Good, < 5% = Poor |
| **Interest Coverage** | EBIT / Interest Expense | > 5x = Safe, 2–5x = Adequate, < 2x = Stressed |

### Scoring Logic

```typescript
function scoreFinancialHealth(data: FundamentalData, financials: FinancialStatements): number {
  let score = 50;

  // Debt-to-Equity
  const de = calculateDebtToEquity(financials);
  if (de !== null) {
    if (de < 0.3) score += 15;         // Very low debt
    else if (de < 0.8) score += 8;     // Conservative
    else if (de > 2.0) score -= 15;    // High leverage
    else if (de > 1.5) score -= 8;     // Moderate leverage
  }

  // Current Ratio
  const cr = calculateCurrentRatio(financials);
  if (cr !== null) {
    if (cr > 2.5) score += 10;
    else if (cr > 1.5) score += 5;
    else if (cr < 0.8) score -= 12;
    else if (cr < 1.0) score -= 6;
  }

  // Free Cash Flow
  const fcf = getFreeCashFlow(financials);
  if (fcf !== null) {
    if (fcf > 0) score += 10;          // Positive FCF
    else score -= 15;                   // Negative FCF
  }

  // ROE
  if (data.roe) {
    if (data.roe > 0.25) score += 10;
    else if (data.roe > 0.15) score += 5;
    else if (data.roe < 0.05) score -= 8;
    else if (data.roe < 0) score -= 15;
  }

  return Math.max(0, Math.min(100, score));
}
```

## 6. Technical Momentum Score (25%)

### Indicators Used

| Indicator | Parameters | How to Score |
|---|---|---|
| **SMA Trend** | SMA(20), SMA(50), SMA(200) | Price > SMA(200) = Bullish, Golden Cross = Very Bullish |
| **RSI** | RSI(14) | < 30 = Oversold (contrarian bullish), > 70 = Overbought (contrarian bearish) |
| **MACD** | MACD(12, 26, 9) | Signal line crossover = trend change |
| **Volume Trend** | 20-day avg vs current | Above average on up days = Bullish |
| **52-Week Range** | Current vs High/Low | Near high = Strength, Near low = Weakness or Value |
| **Bollinger Bands** | BB(20, 2) | Below lower band = Oversold, Above upper = Overbought |

### Scoring Logic

```typescript
function scoreTechnicalMomentum(data: TechnicalData): number {
  let score = 50;

  // SMA Alignment
  if (data.price > data.sma200 && data.sma50 > data.sma200) {
    score += 15; // Bullish alignment
  } else if (data.price < data.sma200 && data.sma50 < data.sma200) {
    score -= 15; // Bearish alignment
  }

  // Golden Cross / Death Cross
  if (data.sma50CrossedAbove200) score += 10;  // Golden Cross
  if (data.sma50CrossedBelow200) score -= 10;  // Death Cross

  // RSI
  if (data.rsi14) {
    if (data.rsi14 < 25) score += 10;        // Deeply oversold (contrarian buy)
    else if (data.rsi14 < 30) score += 5;    // Oversold
    else if (data.rsi14 > 80) score -= 10;   // Deeply overbought
    else if (data.rsi14 > 70) score -= 5;    // Overbought
  }

  // MACD
  if (data.macdCrossover === "bullish") score += 8;
  if (data.macdCrossover === "bearish") score -= 8;

  // 52-Week Position
  const range52w = (data.price - data.low52w) / (data.high52w - data.low52w);
  if (range52w > 0.9) score += 5;    // Near 52w high (strength)
  if (range52w < 0.1) score -= 5;    // Near 52w low (weakness)

  return Math.max(0, Math.min(100, score));
}
```

## 7. Final Score Calculation

```typescript
interface DimensionScore {
  valuation: number | null;
  growth: number | null;
  financialHealth: number | null;
  technicalMomentum: number | null;
}

interface SignalResult {
  score: number;
  signal: "Strong Buy" | "Buy" | "Hold" | "Sell" | "Strong Sell";
  confidenceRange: string;        // e.g., "65–80%"
  availableDimensions: number;    // How many of 4 dimensions had data
  breakdown: DimensionScore;
  dataCompleteness: string;       // "Full" | "Partial" | "Insufficient"
  generatedAt: string;            // ISO 8601 timestamp
}

function calculateFinalSignal(scores: DimensionScore): SignalResult {
  const availableScores = Object.values(scores).filter(s => s !== null) as number[];

  // Require at least 2 dimensions to generate a signal
  if (availableScores.length < 2) {
    return {
      score: 0,
      signal: "Hold",
      confidenceRange: "N/A",
      availableDimensions: availableScores.length,
      breakdown: scores,
      dataCompleteness: "Insufficient",
      generatedAt: new Date().toISOString(),
    };
  }

  // Equal weight redistribution across available dimensions
  const weightedScore = availableScores.reduce((sum, s) => sum + s, 0) / availableScores.length;
  const roundedScore = Math.round(weightedScore);

  // Determine signal
  let signal: SignalResult["signal"];
  if (roundedScore >= 80) signal = "Strong Buy";
  else if (roundedScore >= 65) signal = "Buy";
  else if (roundedScore >= 45) signal = "Hold";
  else if (roundedScore >= 25) signal = "Sell";
  else signal = "Strong Sell";

  // Determine confidence range based on dimensional agreement
  const directions = availableScores.map(s => (s >= 55 ? "bullish" : s <= 45 ? "bearish" : "neutral"));
  const bullishCount = directions.filter(d => d === "bullish").length;
  const bearishCount = directions.filter(d => d === "bearish").length;
  const dominantCount = Math.max(bullishCount, bearishCount);

  let confidenceRange: string;
  if (dominantCount === availableScores.length) confidenceRange = "70–85%";
  else if (dominantCount >= availableScores.length - 1) confidenceRange = "55–70%";
  else confidenceRange = "35–50%";

  return {
    score: roundedScore,
    signal,
    confidenceRange,
    availableDimensions: availableScores.length,
    breakdown: scores,
    dataCompleteness: availableScores.length === 4 ? "Full" : "Partial",
    generatedAt: new Date().toISOString(),
  };
}
```

## 9. Edge Cases & Guardrails

### When NOT to Generate a Signal
1. **IPO stocks (< 1 year of trading data):** Not enough history for meaningful technical analysis. Display: "Insufficient historical data for signal generation."
2. **Penny stocks (price < $1):** Highly volatile and often manipulated. Display: "Signal generation is not available for stocks below $1."
3. **Negative earnings:** P/E becomes meaningless. Skip P/E in valuation score.
4. **Extreme outlier P/E (> 500 or < 0):** Likely distorted by one-time events. Cap at sector median × 3.
5. **Missing fundamental data:** If a company hasn't filed recent financials (e.g., OTC stocks), do not generate a signal.
6. **Market closed / after hours:** Technical signals use closing prices only. Clearly label data as "As of market close [date]."

### Maximum Signal Change Rate
- A signal can change at most **once per trading day**
- This prevents whipsaw signals on intraday volatility
- Signal history is stored per stock for transparency
